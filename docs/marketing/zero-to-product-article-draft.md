事情是这样的。

我是一名会计专业的大四学生。

零编程基础。没学过计算机。没写过代码。连 HTML 长什么样都是上个月才第一次认真看的。

但我刚刚上线了一个 AI 新闻雷达。

24 小时自动更新，抓取全球 AI 动态，AI 强相关过滤，中英双语，GitHub Pages 部署。不用服务器，不用数据库，不用花一分钱。

全程没写一行代码。

不是「我用 AI 辅助写代码」，是真的、一行都没写。

我知道你在想什么。

又来了，又一个标题党。

肯定是用了什么无代码工具拖拽出来的。

或者就是套了个模板。

我要是你，我也不信。

所以我把整个过程完整拆开给你看。一个零基础的人，怎么用 AI，从零搭出一个真实可用的产品。



先说一下我的起点。

会计专业，大四，会的技能包括做账、报税、审计。不会的技能包括 Python、JavaScript、HTML、CSS、Git、GitHub、RSS、OPML、JSON、API。以上全部不会。

但我有一个优势。

我敢试。

我不知道你们有没有这种感觉，你面对一个完全陌生的领域，光是看着那些术语就觉得头晕。我第一次打开 GitHub 的时候，连 Repository 是什么意思都不知道。

但我当时的想法很简单。

我不会写代码，但我可以说人话。AI 听得懂人话就行。



我打开 Claude Code，打了这么一段。

「我想做一个 AI 新闻聚合网站，自动抓取多个信息源，过滤掉不相关的内容，显示最近 24 小时的 AI 动态。不用服务器，部署在 GitHub Pages 上。」

Claude Code 回了我一堆东西。

说实话我当时一半没看懂。

但我没管，我继续跟它聊。像跟一个懂技术的朋友聊天一样。

「这个我不懂，你帮我解释一下。」

「能不能换个更简单的方式。」

「这个方案太复杂了，有没有省事的办法。」

就这样来回聊了几轮，它帮我定下了整个技术方案。Python 脚本抓数据，GitHub Actions 定时运行，静态 JSON 存结果，纯 HTML 页面展示。零服务器，零数据库，零成本。

我寻思了一下我没寻思明白，但听起来挺合理的。

那就开干。

![AI News Radar AI 强相关模式](../illustrations/zero-to-product/01-radar-ai-mode.png)

接下来的事情，比我预想的顺利，也比我想象的坎坷。

顺利的地方在于，Claude Code 真的能干活。我说帮我写一个抓 RSS 的脚本，它就写了。我说这个页面太丑了帮我重新设计一下，它就重新设计了。我说这里能不能加个搜索功能，它就加了。

我跟你说，这种感觉太爽了。

就像一个不会做饭的人，突然有了一个米其林大厨当助手。你只需要说我想吃红烧肉，他就帮你做好了。你不用知道火候怎么控制，酱油放多少，肉要炖多久。

但坎坷的地方在于，AI 不是完美的。



有一次，它帮我写了一个过滤 AI 相关内容的逻辑，我觉得应该没问题，就部署上去了。

结果第二天打开网页一看。

OpenAI 发布新模型的新闻被过滤掉了。

被过滤掉了。

你敢信？？？

我查了半天，发现是因为那篇文章的标题里没有明确出现「AI」这个词，写的是「Introducing our latest model」。它的过滤逻辑只认关键词，不懂语义。

我当时就愣住了。

这不是 AI 吗，怎么连 AI 相关的新闻都认不出来。

后来我加了一层语义评分，不只是关键词匹配，还加了信息源信任度权重。官方 AI 源的新闻自动加分，聚合站的新闻要过更严格的阈值。低于 0.65 分的进候审区，高于的才进 AI 精选。

这个过程教会我一件事。

AI 帮你做东西很快。但要做得对，你得自己验收。

![AI News Radar 全量模式](../illustrations/zero-to-product/02-radar-all-mode.png)

还有一个很搞的事情。

最开始我选了大概二十多个信息源，觉得越多越好。源多了覆盖就全了嘛。

结果跑了一天，生成出来的 JSON 文件十几兆。

我一开始还挺高兴的，觉得数据多就是好。

直到我打开网页。

加载了四十秒。

四十秒。

我盯着那个白屏看了四十秒。

后来 Claude Code 帮我分析，发现有接近三分之二的内容是重复的。不同聚合站之间互相抄，同一个新闻被转了五六遍。有些源看起来更新很多，其实跟 AI 只沾了 5% 的边。

太离谱了。

然后我们一起做了三件事。去重逻辑，信源重叠检测，源健康状态监控。不是所有源都值得长期追踪，得先判断哪些是千里马，哪些是噪音。

这一轮折腾完，加载时间从四十秒降到了不到一秒。

几百条高质量的 AI 信号，按站点分组，中英双语标题，一键搜索，源健康面板。

第一次感觉自己做出了一个真正有用的东西。



很多人以为零基础做产品的最大障碍是技术。

不是。

技术问题，AI 可以帮你解决。真的，几乎所有技术问题 AI 都能帮你搞定，只要你问得够清楚。

最难的是两件事。

第一，你不知道你不知道什么。

你不会 Python，你甚至不知道 Python 能干嘛。你不知道什么是 RSS，不知道 GitHub Pages 是什么，不知道静态网站和动态网站的区别。你根本提不出正确的问题。你得先花时间了解有哪些可能性，然后才能指挥 AI 去做。

第二，做决策。

AI 可以给你十个方案。但选哪个，它不能替你决定。

我当时在 RSSHub 和自建 RSS 之间纠结了很久。RSSHub 成熟稳定，但我想要更灵活的控制。自建灵活，但坑更多。最后选了自建，不是因为我懂技术，是因为我想明白了自己要什么。我想要的是一个活的、能持续进化的系统，不是一次性搭完就不碰的静态页面。

你如果关注这个领域的话，会发现一个很有意思的现象。

技术门槛在消失，但决策门槛在变高。

门槛从「怎么写代码」变成了「怎么把问题想清楚」。后面这个，AI 暂时帮不了你。

![AI News Radar 搜索功能](../illustrations/zero-to-product/03-radar-search.png)

现在这个东西长什么样。

打开网页，一个很干净的界面。顶部数据概览，多少条信号，多少个站点，多少个来源。中间搜索框和分类筛选，你可以只看 AI 强相关的内容，也可以看全量。每条新闻有来源标签和发布时间，点一下就能跳过去。

往下拉，WaytoAGI 开源社区的更新日志。

再往下，信息源健康状态，哪些源正常，哪些挂了。

整个页面没有广告，没有付费墙，纯静态，加载飞快。

每 30 分钟自动更新一次，全自动运行。



我说「理论上」是因为我自己还没完全跑通 GitHub Pages 的自动部署，GitHub 需要账号，我暂时还没弄。目前本地跑通了完整流程，网页在本地能打开，数据能自动刷新。就差最后一步推上去。

但核心的东西都跑通了。



说真的，我做这个东西，最开始纯粹是为了自己。

我不想每天花一个小时刷新闻了。我想把这个时间省下来，去做更有意思的事。

但做完之后，我在想另一件事。

像我这样的人应该很多。不是程序员，不会写代码，但对 AI 真的很好奇。想追前沿，又不知道怎么高效地获取信息。每天被各种标题党和营销号轰炸，分不清什么是真正重要的更新。

如果这个工具能帮到他们呢。

如果打开一个网页，就能看到过去 24 小时 AI 领域真正值得关注的事情。不用装任何东西，不用注册任何账号，不用学任何技术。

坦率的讲，这就是我想做的事。

磨平一些信息差。



我想到一个意象。

伯乐相马。

古人说千里马常有，而伯乐不常有。现在 AI 信息也是这样，好内容到处都有，但缺的是一个帮你从几千条噪音里挑出那几十条真正值得看的东西的人。

这个新闻雷达，就是在做伯乐的事。

不是替你判断什么重要，而是帮你筛掉那些明显不重要的。把信噪比从 1 比 100 拉到大概 1 比 5。剩下的 5 条，你自己决定看不看。



我自己也还在摸索，可能有些想法还不成熟，技术方案也不是最优的。但我始终坚信一件事。

AI 时代，最重要的不是你会什么，而是你敢不敢试。

我会计大四，一行代码没写过，照样搭出了一个能用的 AI 新闻雷达。

屏幕前的你，不管你是什么背景，只要你想做，你真的可以做出来。

工具都在那了。就差你迈出第一步。



我最近老想起小时候玩积木。

你不需要会做积木，你只需要知道你想搭什么。城堡也好，飞机也好，你知道大概长什么样就行。剩下的就是一块一块往上拼。

AI 让所有人都变成了可以搭积木的人。

你不知道怎么做积木，但你知道你想搭什么。

这就够了。
